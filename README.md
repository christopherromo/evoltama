# Evoltama

##### Created by Kristen Poteat and Christopher Romo
##### CS 3110 Programming the Mobile Web : Spring 2025

---

**Evoltama** is a RPG-style game that centers around the Evolisks, magical creatures that have special powers. The game is built solely with JavaScript, HTML, and CSS. The game features overworld rendering, story progression, a turn-based battle system, and more. Designed to be a browser-based game, **Evoltama** aims to provide players with a quick and fun experience.

*This game was mainly created by using the following tutorial on YouTube:*  
<https://www.youtube.com/playlist?list=PLcjhmZ8oLT0r9dSiIK6RB_PuBWlG1KSq_>

*All in-game image assets were created by Kristen Poteat and Christopher Romo.*

## Technologies 💻

   - JavaScript
   - HTML
   - CSS

## Features 📄

   - **Overworld Rendering:** Players can explore the world of **Evoltama**, featuring locations, characters, and interactive objects like the Evolisk Stone.

   - **Progression:** Various interactions progress the story, allowing for different areas and new dialogue to be unlocked. Game progression can be saved though the pause menu.

   - **Turn-Based Battle System:** Players can form their team of Evolisks, and battle other Evolisks. On their turn, the player has the choice of attacking, using an item, and swapping out their current Evolisk for another Evolisk.

   - **Evolisk Mutation:** Throughout the game, Evolisks are awarded with XP upon winning a battle. Certain Evolisks can mutate, changing their appearance and leveling up their stats.

   - **Original Assets:** Everything seen in game (with the exception of certain HUD elements), including the maps, characters, and interactive objects, is an original asset specifically created for this game!

## Reflection 💭

**The Process:** To start, we were given the task of creating any website as a team. My partner, Kristen Poteat, recommended we make a game for our website. We wanted to explore JavaScript, HTML, and CSS further than what we learned in class. We found a tutorial (linked above) on YouTube for a RPG-style game that is solely built with JavaScript, HTML, and CSS. All credit for the game's mechanics goes to the creator of the original game, Drew Conley. The project was completed in 4 2-week sprints. The first two sprints focused on following the YouTube guide for each component. The last two sprints focused on creating assets, creating the story, and implementing the actual game.

**Key Takeaways:** This project was less about creating an original codebase and more about learning *what* we can do with JavaScript, HTML, and CSS. For example, we learned many things about JavaScript, including various ways to write functions and store information. We learned about the HTML canvas element, which is where our game lives. We also learned many things about CSS, including variables and animation. On top of that, this project also taught us some things about game development, which neither of us had experience in. We learned how to create assets like sprite sheets and maps, which I created with Pixquare, a pixel art editor. We also learned more mechanical components like the actual game loop, which is in `Overworld.js`. Overall, we were able to create a charming game by using various resources.
